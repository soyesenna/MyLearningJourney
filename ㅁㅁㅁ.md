**1. 데이터 통신(Data Communication)의 의미**

  

데이터 통신이란, **두 개 이상의 장치(컴퓨터, 단말기 등) 간에 데이터를 전송하고 그 정보를 공유하는 것**을 의미한다. 과거에는 음성 중심의 아날로그 통신이 주류였으나, 현대에는 디지털 기술의 발달로 인해 음성, 영상, 텍스트 등 다양한 형태의 정보를 디지털 데이터로 변환하여 전달하는 통신이 주를 이룬다. 즉, 데이터 통신은 단순한 신호의 교환이 아니라 **정보의 효율적 교환**을 목적으로 한다.

  

**2. 데이터 통신의 기본 특성과 데이터 통신 시스템의 5가지 구성요소**

  

**(1) 데이터 통신의 기본 특성**

1. **전달(Delivery)**: 전송된 정보가 올바른 목적지에 전달되어야 한다.

2. **정확성(Accuracy)**: 전송 과정에서 오류 없이 정확하게 전달되어야 한다.

3. **적시성(Timeliness)**: 필요한 시점에 정보를 전달받아야 한다(지연 최소화).

4. **신뢰성(Reliability)**: 전송 과정에서 재전송, 오류 제어 등으로 데이터 손실 및 오류를 최소화해야 한다.

  

**(2) 데이터 통신 시스템의 5가지 구성요소**

1. **송신기(Sender)**: 데이터를 생성하고 전송을 시작하는 장치(예: 컴퓨터, 스마트폰).

2. **수신기(Receiver)**: 전송된 데이터를 받아들이는 장치(예: 서버, 프린터).

3. **메시지(Message)**: 송신기가 전달하고자 하는 실제 정보(텍스트, 이미지, 오디오, 영상 등).

4. **전송 매체(Transmission Medium)**: 메시지가 이동하는 물리적 경로(유선 케이블, 광섬유, 무선 등).

5. **프로토콜(Protocol)**: 데이터 통신에 필요한 규약이나 약속(전송 방식, 오류 제어, 데이터 형식 등).

  

**3. 데이터 통신에서의 데이터 표현 방식**

  

데이터 통신에서 주고받는 정보는 일반적으로 **디지털 형태** 혹은 **아날로그 형태**로 표현된다.

• **아날로그 신호(Analog Signal)**

- 연속적인 파형의 형태(음성, 음악, 영상 신호 등).

- 예: 라디오 방송의 전파, 전화 선로를 통한 음성 신호.

• **디지털 신호(Digital Signal)**

- 불연속적인 0과 1의 이진 비트로 표현.

- 예: 컴퓨터 내부 데이터, 인터넷 패킷 전송 데이터.

  

현대의 대부분의 통신은 디지털 기반으로 이루어지지만, 여전히 아날로그 신호를 수집·변환하는 과정(ADC: Analog to Digital Conversion 등)이 필요한 경우가 있다.

  

**4. 데이터 전송 방식**

  

데이터 전송 방식에는 크게 다음과 같은 형태가 있다.

1. **직렬 전송(Serial Transmission)**

- 데이터를 한 비트씩 순차적으로 전송하는 방식.

- 긴 거리 전송에 적합하고, USB, RS-232 등이 예시이다.

2. **병렬 전송(Parallel Transmission)**

- 여러 비트를 동시에 전송하는 방식.

- 짧은 거리에서 빠른 속도로 데이터를 전송 가능하지만, 케이블 비용이 높고, 장거리 전송에는 부적합하다.

3. **동기식(Synchronous) 전송**과 **비동기식(Asynchronous) 전송**

- 동기식 전송: 송·수신 측이 공통의 클록 신호(시간 기준)를 공유하며, 대량의 데이터를 블록 단위로 전송.

- 비동기식 전송: 전송 단위 블록(문자 단위 등) 시작·종료 신호를 통해 동기를 맞추며, 주로 저속 통신이나 문자 전송에 사용.

  

**5. 네트워크의 개념과 채널(Channel)에 대한 설명**

  

**(1) 네트워크의 개념**

  

네트워크(Network)란, **데이터 통신 장치들을 유기적으로 연결하여 자원을 공유하고 상호 간의 정보를 주고받을 수 있도록 만든 통신 시스템**을 의미한다. 인터넷은 이러한 개별 네트워크들이 전 세계적으로 연결된 거대한 네트워크의 집합체라고 할 수 있다.

  

**(2) 채널(Channel)**

  

채널은 **데이터가 실제로 전송되는 물리적 경로 혹은 논리적 경로**를 의미한다.

• **물리적 채널**: 케이블, 광섬유, 무선 링크처럼 실제 전송에 쓰이는 매체.

• **논리적 채널**: 동일한 물리적 채널을 여러 사용자 혹은 프로세스가 공유할 때의 가상 경로(TDM, FDM 등).

  

**6. 네트워크 평가 기준**

  

네트워크 성능 및 품질은 보통 다음과 같은 요소로 평가된다.

1. **대역폭(Bandwidth)**: 단위 시간당 전송할 수 있는 데이터의 최대량(주로 bps로 표시).

2. **지연(Latency, Delay)**: 데이터가 송신에서 수신까지 도달하는 데 걸리는 시간. 전파 지연, 전송 지연, 처리 지연 등이 복합적으로 작용한다.

3. **손실률(Error Rate/Ber)**: 전송 중 발생하는 비트 오류의 비율.

4. **신뢰성(Reliability)**: 네트워크가 얼마나 안정적으로 운영되고 장애가 적은지 나타내는 척도.

5. **확장성(Scalability)**: 사용자 수나 트래픽 양이 증가했을 때 네트워크가 무리 없이 확장될 수 있는지 평가.

  

**7. 네트워크 토폴로지(Topology)의 종류와 특징**

1. **버스(Bus) 토폴로지**

- 모든 노드가 하나의 공용 선(bus)에 연결된 형태.

- 설치가 간단하고 저비용이나, 공용 선에 장애가 생기면 전체가 영향받는다.

2. **스타(Star) 토폴로지**

- 중앙에 허브(혹은 스위치, 라우터 등)가 있고, 모든 노드가 중앙 장치와 직접 연결.

- 장애 발생 시 중앙 장치에서 쉽게 문제 파악 가능. 하지만 중앙 장치에 부하가 집중되거나 장애가 나면 전체 네트워크가 마비된다.

3. **링(Ring) 토폴로지**

- 각 노드가 양 옆 노드와 연결되어 고리(ring)를 형성.

- 데이터가 순환 형태로 전송되고, 특정 지점에 장애가 발생하면 반대 방향으로 데이터가 전송될 수 있는 이중 링 방식을 쓰기도 한다.

4. **메시(Mesh) 토폴로지**

- 여러 노드가 서로 직접 연결되어 있는 망형 구조.

- 경로가 다원화되어 안정적이나, 설치와 비용이 많이 든다.

- 완전 연결(Full Mesh)과 부분 연결(Partial Mesh)로 구분한다.

5. **트리(Tree) 토폴로지**

- 트리 구조(계층 구조)로 확장한 형태.

- 대규모 네트워크에서 계층적인 관리가 용이하나, 특정 계층에 장애가 발생하면 하위 노드들이 영향을 받을 수 있다.

  

**8. 네트워크 유형**

1. **LAN(Local Area Network)**

- 좁은 범위(사무실, 건물, 캠퍼스 등)에서 컴퓨터와 장치를 연결.

- 데이터 전송 속도가 빠르고, 관리가 상대적으로 쉽다.

2. **MAN(Metropolitan Area Network)**

- 도시나 군 정도의 중간 범위를 커버하는 네트워크.

- LAN과 WAN의 중간 규모이며, 케이블 TV망, 지역 인터넷 서비스 등이 예시에 해당.

3. **WAN(Wide Area Network)**

- 국가 전체 혹은 전 세계를 연결하는 광역 네트워크.

- 주로 ISP(인터넷 서비스 제공업체)가 관리하며, 인터넷 자체가 대표적인 WAN이다.

4. **PAN(Personal Area Network)**

- 개인이 사용하는 기기(스마트폰, 노트북, 웨어러블 등) 간의 근거리 통신을 위한 네트워크.

- 블루투스, ZigBee 등이 대표 예시.

  

**9. 교환망의 개념 및 회선 교환망과 패킷 교환망의 특징**

  

**(1) 교환망(Switched Network)의 개념**

  

교환망이란, **네트워크 내의 교환기(스위치)가 데이터 전송 경로를 설정하고, 경로를 따라 데이터를 전송하는 방식**을 사용하는 네트워크이다.

  

**(2) 회선 교환망(Circuit Switching)**

• **특징**: 통신을 시작하기 전에 송·수신 노드 간에 물리적으로 전용 경로(회선)를 설정하고, 통신이 종료될 때까지 해당 경로를 독점한다.

• **장점**: 경로가 전용되어 있으므로 지연이 거의 없고, 실시간 음성·영상 통신에 적합.

• **단점**: 경로가 독점 상태이므로 비효율적(통화 중 무음 구간에도 회선이 점유).

  

**(3) 패킷 교환망(Packet Switching)**

• **특징**: 전송할 데이터를 일정한 크기의 패킷 단위로 나누어, **네트워크의 여러 경로를 통해 라우팅**하면서 전송한다.

• **장점**: 회선 독점이 없으므로 네트워크 자원을 효율적으로 사용 가능, 트래픽이 많은 인터넷 환경에 적합.

• **단점**: 패킷 우회 경로 등에 의해 지연이 발생할 수 있으며, 패킷 순서가 뒤바뀔 가능성도 존재(재조립 필요).

  

**10. 프로토콜(Protocol)의 개념과 구성요소, 그리고 표준의 필요성**

  

**(1) 프로토콜의 개념**

  

프로토콜은 **데이터를 송·수신하기 위해 필요한 일련의 규칙, 약속, 절차**를 의미한다. 간단히 말하면, 서로 다른 기기나 프로그램 간에 **어떻게 데이터를 교환할지**를 정해 놓은 통신 규칙이다.

  

**(2) 프로토콜의 구성요소**

1. **구문(Syntax)**: 전송되는 데이터의 형식, 부호화 방식, 신호 레벨 등.

2. **의미(Semantics)**: 각 제어 정보(명령, 응답 등)의 의미와 이에 대한 동작 방식.

3. **타이밍(Timing)**: 전송 속도, 순서 제어, 흐름 제어 등 시간적 이슈.

  

**(3) 표준의 필요성**

• 서로 다른 기기나 소프트웨어 간 호환성을 보장하여, **상호 운용성(Interoperability)**을 향상시킨다.

• 범국제적으로 통용되는 약속을 확립하여 기술 개발과 보급을 촉진한다. 예를 들어, **OSI 참조 모델, TCP/IP 프로토콜 스택** 등이 대표적 표준이다.
